MiniSimmy2
Molecular Evolution Toy

A simplified, visually-driven evolutionary simulation focused on time-dependent environments and modular body plans.

Explore Concept View Roadmap
THE VISION

A Lighter, More Focused Evolution Toy

MiniSimmy2 is a spiritual successor to the original Mini Simmy, but built around a different core idea: time-dependent environmental pressure and modular creature bodies.

It draws inspiration from the module system and day/night mechanics in the main Python Modular Life Sim, but strips away macros, spring physics, and complex internal ecosystems to stay focused and visually clear.

Core Philosophy

  • • Visual clarity first
  • • Strong environmental rhythm (day/night)
  • • Simple but meaningful module system
  • • Evolution through body plan variation
  • • Keep scope tight and delightful
VISUAL IDENTITY

Molecule Creatures

Basic Forager
Yellow core + 3 modules
Specialized
Denser module cluster
Minimalist
Fewer but larger modules

Creatures are visualized as dense molecular clusters: a bright yellow core (energy/identity) surrounded by smaller colored nodules (modules). The visual density and color distribution should immediately communicate the creature’s “body plan” and specialization.

CORE MECHANICS

Day/Night & Environmental Pressure

Day / Night Cycle

A simple global cycle that affects lighting, creature energy drain, and resource effectiveness. The world shifts between warm (day) and cool (night) tones.

Two Resource Zones

  • Green Zone — Strong during day, damages creatures at night
  • Blue Zone — Strong during night, damages creatures during day
MODULE SYSTEM

Attachable Colored Components

Creatures are built from a central yellow core + multiple small colored modules. Different module types provide passive or active effects. New modules can appear through reproduction.

Harvester
Better resource intake from zones
Sensor
Better detection of resources or threats
Mover
Increased movement speed
Resistant
Reduced damage from harmful zones
REPRODUCTION & EVOLUTION

Module Inheritance + Novelty

When creatures reproduce, offspring inherit most of the parent’s modules. There is a chance to gain one completely new random module type. This creates gradual body plan evolution and occasional innovation.

DEVELOPMENT ROADMAP

Phased Build Plan

Phase 0
Foundation
Canvas + day/night cycle + two resource zones + basic molecule creatures (yellow core + simple modules)
Phase 1
Modules & Visuals
Distinct module types with visual clustering, basic passive effects, improved creature rendering
Phase 2
Reproduction & Inheritance
Energy-based or time-based reproduction, module inheritance, chance for new random modules on birth
Phase 3
Behavior & Evolution
Simple module-driven behaviors, basic selection pressure, population stats panel
Phase 4
Polish & Integration
Sound design, refined visuals, website integration, optional evolution tracking dashboard
TECHNICAL APPROACH

Single-File HTML + p5.js

Like the original Mini Simmy, MiniSimmy2 will be built as a clean, self-contained single HTML file using p5.js for rendering and Tailwind for UI. This keeps it easy to share, iterate, and embed.
